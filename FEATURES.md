# Note Summary 功能清单

> 版本：v1.3 | 更新日期：2026-04-26

---

## 一、用户认证

| 功能 | 说明 |
|------|------|
| 用户注册 | 用户名 + 密码 + 确认密码，密码 ≥ 6 位，用户名唯一性校验 |
| 用户登录 | 用户名 + 密码登录，返回 JWT Token（有效期 24 小时） |
| 修改密码 | 需验证原密码，新密码 + 确认新密码，密码 ≥ 6 位 |
| 退出登录 | 清除本地 Token，跳转登录页 |
| 自动鉴权 | 所有接口需 Bearer Token 认证，未登录自动跳转登录页 |
| Token 过期处理 | 401 响应自动清除 Token 并跳转登录页 |
| JWT 安全验证 | 显式启用过期时间验证（verify_exp），默认密钥启动时告警 |
| 数据隔离 | 每个用户只能访问自己的笔记、分类数据 |

## 二、笔记管理

| 功能 | 说明 |
|------|------|
| 创建笔记 | 输入原始内容，AI 自动整理为结构化笔记 |
| AI 自动整理 | 自动生成标题、摘要、结构化内容、关键词 |
| AI 自动分类 | AI 建议分类，自动创建或匹配已有分类 |
| AI 自动打标签 | AI 建议标签，自动创建或匹配已有标签，多对多关联 |
| 自动向量化 | 将标题+内容+摘要生成 embedding 向量，用于语义搜索 |
| 内容类型识别 | AI 自动判断类型：生活/感想/知识/待办/灵感/工作 |
| 笔记列表 | 分页展示，支持按分类、类型、关键词、收藏状态筛选 |
| 笔记详情 | 展示标题、摘要、结构化内容（Markdown 渲染）、标签、类型、时间 |
| 查看原始输入 | 折叠展示用户输入的原始内容 |
| 编辑笔记 | 支持修改标题、内容、摘要、分类、标签、来源类型等字段，就地编辑模式 |
| 软删除 | 删除笔记时标记为已删除而非物理删除，可恢复 |
| 回收站 | 独立的回收站页面，展示已删除笔记，支持恢复和彻底删除 |
| 恢复笔记 | 从回收站恢复笔记到原分类（原分类已删除则归入未分类） |
| 彻底删除 | 从回收站永久删除笔记，同时清理向量数据，不可恢复 |
| 自动清理 | 回收站中超过 30 天的笔记自动永久删除 |
| 收藏笔记 | 一键收藏/取消收藏，侧边栏「已收藏」入口筛选查看 |
| 置顶笔记 | 一键置顶/取消置顶，置顶笔记始终排在列表最前面 |
| 智能排序 | 笔记列表按 置顶 > 收藏 > 创建时间倒序 排列 |
| Markdown 预览 | 输入时实时渲染 Markdown 格式 |

## 三、AI 功能

| 功能 | 说明 |
|------|------|
| 内容整理（organize） | 将原始内容整理为标题+结构化内容+摘要+分类+标签+关键词 |
| AI 预览 | 保存前可预览 AI 整理结果 |
| 深度调研（research） | 基于笔记内容或指定主题，AI 生成深度扩展知识报告 |
| 调研 SSE 进度推送 | 调研过程实时推送进度（准备→调研中→保存→完成），前端显示分阶段状态和动画 |
| 调研报告保存 | 调研内容自动保存到笔记的 research_content 字段 |
| 知识扩展 | 调研包含概念解释、知识延伸、实际案例、学习建议 |
| 多模型支持 | 支持文本嵌入模型和多模态嵌入模型 |

## 四、搜索

| 功能 | 说明 |
|------|------|
| 全文搜索（FTS） | SQLite FTS5 / PostgreSQL ILIKE，匹配标题、内容、摘要、关键词 |
| 语义搜索 | 基于 embedding 向量余弦相似度，理解语义而非关键词 |
| 混合搜索（默认） | 先全文搜索，无结果自动降级到语义搜索 |
| 搜索结果排序 | 全文搜索按匹配优先级排序（标题 > 关键词 > 摘要 > 内容） |
| 分页支持 | 搜索结果支持 limit/offset 分页 |
| 搜索过滤 | 搜索结果自动过滤已删除的笔记 |

## 七、双向链接

|------|------|
| [[标题]] 语法 | 在笔记内容中使用 `[[笔记标题]]` 建立笔记间链接 |

| Wiki 链接渲染 | 已匹配的链接渲染为蓝色可点击标签，未匹配的显示灰色虚线 |

## 十、技术特性

| 特性 | 说明 |
|------|------|
| JWT 认证 | HS256 算法，Token 24 小时过期，显式过期验证 |
| bcrypt 密码哈希 | 密码安全存储 |
| 双数据库支持 | SQLite（开发）+ PostgreSQL（生产）自动切换 |
| 数据库连接池 | PostgreSQL 连接池（QueuePool），TCP 保活 + 连接回收 + LIFO 复用 + 后台心跳检测 |
| pgvector 向量检索 | PostgreSQL 下使用原生向量距离操作符 |
| NumPy 内存索引 | SQLite 下使用矩阵运算实现高效向量搜索 |
| FTS5 全文检索 | SQLite 下使用虚拟表实现全文搜索 |
| 前后端分离 | Vue 3 + FastAPI，API 驱动 |
| CORS 支持 | 跨域配置，支持开发代理 |
| 静态文件托管 | 后端自动托管前端构建产物 |
| 线程安全 | VectorIndex 单例，线程锁保护 |
| 数据库自动迁移 | 启动时自动检测并添加新列（user_id、is_deleted、deleted_at、is_starred、is_pinned） |
| 后台异步任务 | 向量删除在后台线程执行，不阻塞主请求 |
| 后台定时清理 | 守护线程每 24 小时自动清理回收站过期笔记（30 天） |
| ECharts 图表 | 按需引入 ECharts，lazy import 统计页面 |
| SSE 实时推送 | AI 调研使用 Server-Sent Events 流式推送进度，前端 ReadableStream 解析 |
| 双向链接引擎 | 正则提取 `[[标题]]` + 精确匹配 + NoteLink 关联表 + 反向引用查询 |
| 自定义 Markdown 插件 | markdown-it inline rule 插件实现 `[[标题]]` 渲染为可点击链接 |
