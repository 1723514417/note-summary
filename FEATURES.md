# Note Summary 功能清单

> 版本：v1.0 | 更新日期：2026-04-22

---

## 一、用户认证

| 功能 | 说明 |
|------|------|
| 用户注册 | 用户名 + 密码 + 确认密码，密码 ≥ 6 位，用户名唯一性校验 |
| 用户登录 | 用户名 + 密码登录，返回 JWT Token（有效期 24 小时） |
| 修改密码 | 需验证原密码，新密码 + 确认新密码，密码 ≥ 6 位 |
| 退出登录 | 清除本地 Token，跳转登录页 |
| 自动鉴权 | 所有接口需 Bearer Token 认证，未登录自动跳转登录页 |
| Token 过期处理 | 401 响应自动清除 Token 并跳转登录页 |
| 数据隔离 | 每个用户只能访问自己的笔记、分类数据 |

## 二、笔记管理

| 功能 | 说明 |
|------|------|
| 创建笔记 | 输入原始内容，AI 自动整理为结构化笔记 |
| AI 自动整理 | 自动生成标题、摘要、结构化内容、关键词 |
| AI 自动分类 | AI 建议分类，自动创建或匹配已有分类 |
| AI 自动打标签 | AI 建议标签，自动创建或匹配已有标签，多对多关联 |
| 自动向量化 | 将标题+内容+摘要生成 embedding 向量，用于语义搜索 |
| 内容类型识别 | AI 自动判断类型：生活/感想/知识/待办/灵感/工作 |
| 笔记列表 | 分页展示，支持按分类、类型、关键词筛选 |
| 笔记详情 | 展示标题、摘要、结构化内容（Markdown 渲染）、标签、类型、时间 |
| 查看原始输入 | 折叠展示用户输入的原始内容 |
| 编辑笔记 | 支持修改标题、内容、摘要、分类、关键词等字段 |
| 删除笔记 | 确认后删除，同时清除关联的向量数据 |
| Markdown 预览 | 输入时实时渲染 Markdown 格式 |

---

## 三、AI 功能

| 功能 | 说明 |
|------|------|
| 内容整理（organize） | 将原始内容整理为标题+结构化内容+摘要+分类+标签+关键词 |
| AI 预览 | 保存前可预览 AI 整理结果 |
| 深度调研（research） | 基于笔记内容或指定主题，AI 生成深度扩展知识报告 |
| 调研报告保存 | 调研内容自动保存到笔记的 research_content 字段 |
| 知识扩展 | 调研包含概念解释、知识延伸、实际案例、学习建议 |
| 多模型支持 | 支持文本嵌入模型和多模态嵌入模型 |

## 四、搜索

| 功能 | 说明 |
|------|------|
| 全文搜索（FTS） | SQLite FTS5 / PostgreSQL ILIKE，匹配标题、内容、摘要、关键词 |
| 语义搜索 | 基于 embedding 向量余弦相似度，理解语义而非关键词 |
| 混合搜索（默认） | 先全文搜索，无结果自动降级到语义搜索 |
| 搜索结果排序 | 全文搜索按匹配优先级排序（标题 > 关键词 > 摘要 > 内容） |
| 分页支持 | 搜索结果支持 limit/offset 分页 |

| 功能 | 说明 |

## 七、UI 与交互

| 功能 | 说明 |
|------|------|
| 侧边栏导航 | 收缩/展开侧边栏，hover 时展开显示文字，固定在屏幕左侧不随页面滚动 |
| 路由高亮 | 当前页面导航项自动高亮 |
| Toast 通知 | 全局消息提示（成功/错误），支持自定义显示时长，带淡入淡出动画 |
| 暗色模式 | 支持亮色/暗色主题一键切换，自动检测系统偏好，主题偏好持久化到 localStorage |
| 暗色模式全局适配 | 所有页面（登录、首页、搜索、分类、详情）完整适配暗色模式，包括滚动条、输入框、弹窗等 |
| CSS 变量主题系统 | 54 个语义化 CSS 变量，通过 `[data-theme="dark"]` 选择器切换，所有颜色零硬编码 |
| 响应式布局 | 主内容区自适应填充，视口锁定布局（内容区独立滚动） |
| 来源类型标签 | 不同类型显示不同颜色标签（生活-绿/想法-黄/知识-蓝/待办-粉/创意-紫/工作-橙） |
| 用户信息展示 | 侧边栏底部显示当前登录用户名 |
| 修改密码弹窗 | 模态框形式，支持取消和确认 |

## 八、技术特性

| 特性 | 说明 |
|------|------|
| JWT 认证 | HS256 算法，Token 24 小时过期 |
| bcrypt 密码哈希 | 密码安全存储 |
| 双数据库支持 | SQLite（开发）+ PostgreSQL（生产）自动切换 |
| 数据库连接池 | PostgreSQL 连接池（QueuePool），TCP 保活 + 连接回收 + LIFO 复用 + 后台心跳检测 |
| pgvector 向量检索 | PostgreSQL 下使用原生向量距离操作符 |
| NumPy 内存索引 | SQLite 下使用矩阵运算实现高效向量搜索 |
| FTS5 全文检索 | SQLite 下使用虚拟表实现全文搜索 |
| 前后端分离 | Vue 3 + FastAPI，API 驱动 |
| CORS 支持 | 跨域配置，支持开发代理 |
| 静态文件托管 | 后端自动托管前端构建产物 |
| 线程安全 | VectorIndex 单例，线程锁保护 |
| 数据库自动迁移 | 自动检测并添加新列（如 user_id） |
| 后台异步任务 | 向量删除在后台线程执行，不阻塞主请求 |

## 九、待添加功能

### 🔴 高优先级（核心体验缺失）

| 功能 | 说明 |
|------|------|
| 笔记编辑 | 前端缺少编辑入口，笔记创建后无法在界面上修改内容 |
| 回收站/软删除 | 当前删除为硬删除，误删无法恢复。建议增加软删除 + 回收站 |
| 数据统计面板 | 首页缺少数据概览，如笔记总数、分类分布、最近活跃等 |

### 🟡 中优先级（提升使用价值）

| 功能 | 说明 |
|------|------|
| 笔记关联/双向链接 | 知识库的核心价值在于知识间的关联，目前笔记之间完全孤立，建议支持 `[[双向链接]]` + 反向引用面板 |
| 每日回顾/随机回顾 | 随机展示历史笔记帮助复习，类似 Anki 的间隔复习理念 |
| 收藏/置顶 | 重要笔记无法标记或置顶，找重要内容不方便 |
| 标签管理页面 | 标签只能被动查看，缺少独立的标签管理页面（重命名、合并、删除） |
| 图片上传 | 笔记只能纯文本，无法插入图片 |
| 移动端适配 | 当前 UI 未做移动端响应式，手机上无法良好使用 |

### 🟢 低优先级（锦上添花）

| 功能 | 说明 |
|------|------|
| 笔记导出 | 导出为 Markdown / PDF 格式 |
| 导入功能 | 从其他笔记工具（Notion、Obsidian）批量导入 |
| 版本历史 | 笔记修改历史，支持回滚 |
| 快捷键 | 键盘快捷操作（保存、搜索、新建等） |
| 分享功能 | 生成分享链接，公开分享单篇笔记 |
| 全局搜索 | 类似 Cmd+K 的全局快速搜索弹窗 |
| 批量操作 | 批量删除、批量移动分类 |
| WebSocket 通知 | AI 处理耗时较长时，实时推送进度 |
